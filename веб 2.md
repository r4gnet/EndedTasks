5. Lab: CSRF vulnerability with no defenses
Войдем в предложенный аккаунт, есть форма для изменения почты. При попытке отправки, получаем следующий POST запрос(Перехватив его при помощи burp suit). Отправим запрос в repeater.
Создадим файл с html кодом для проведения CSRF атаки.
Теперь переходим на exploit сервер, и туда в body вставляем этот код. Сохраняем его. Теперь при демонстрации этого, почта пользователя заменяется на нашу.
Теперь ещё раз меняем почту злоумышленника, и выбираем отправить жертве.
